# 贡献指南

感谢你参与 Coco。

## 提交 Issue

提交前请先搜索已有 Issue，避免重复。Bug 报告应尽可能包含最小复现步骤；涉及文档内容时，只提交已脱敏且有权分享的文件。

## 提交资源

workflow、skill 和 template 的正式 manifest 仍在设计中。当前可以通过 Issue 讨论资源需求或提交示例，但请遵守：

- 不包含密钥、账号、内部地址或客户数据；
- 不上传无权再分发的文档和模板；
- 不提交混淆代码、下载器或未经说明的网络请求；
- 包含提示词时，说明所需模型能力和文档数据使用范围；
- 包含文档写入能力时，明确可能修改的内容。

## Pull Request

Pull Request 应保持单一目的，并说明：

- 解决的问题；
- 主要变化；
- 验证方式；
- 是否涉及兼容性、权限或安全边界。

代码型 skill 和任意 JavaScript 执行资源暂不进入官方资源索引。

## 行为准则

请保持专业、友善并尊重不同观点。骚扰、人身攻击、泄露隐私或发布无权公开的信息不会被接受。
